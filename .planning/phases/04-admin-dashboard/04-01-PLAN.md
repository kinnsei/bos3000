    **App Shell:**

    1. Create main.tsx: Render `<App />` which wraps with Providers.

    2. Create providers.tsx:
       - QueryClientProvider with QueryClient (staleTime: 5min, retry: 1, refetchOnWindowFocus: false)
       - ThemeProvider: Reads system preference via matchMedia('prefers-color-scheme: dark'), stores choice in localStorage, toggles `.dark` class on `<html>`. Export useTheme() hook.
       - RouterProvider with the router from router.tsx

    3. Create router.tsx using React Router v7 library mode:
       - `createBrowserRouter` (import from `react-router`, NOT react-router-dom)
       - `/login` route -> LoginPage
       - `/` route -> Layout with children (all lazy-loaded with Suspense + SkeletonPage fallback):
         - index -> Dashboard, /customers -> Customers, /customers/:id -> CustomerDetail
         - /gateways -> Gateways, /cdr -> CDR, /wastage -> Wastage
         - /finance -> Finance, /did -> DID, /compliance -> Compliance
         - /ops -> Ops, /settings -> Settings
       - For now, create placeholder page components (just `<div>Feature Name</div>`) for each route. Feature plans will replace them.

    4. Create layout.tsx using shadcn Sidebar component:
       - Left collapsible sidebar with menu groups per user decision:
         - Monitoring: Overview (icon: LayoutDashboard), CDR (icon: PhoneCall), Wastage (icon: TrendingDown)
         - Management: Customers (icon: Users), Gateways (icon: Network), DID (icon: Hash)
         - Finance: Finance Center (icon: Wallet)
         - System: Compliance (icon: Shield), Operations (icon: Activity), Settings (icon: Settings)
       - All menu labels in Chinese per user decision
       - Top navigation bar: Logo text "BOS3000", spacer, theme toggle button (Sun/Moon icons), notification bell icon (placeholder), admin avatar dropdown (placeholder)
       - Main content area: `<Outlet />` from react-router
       - Sidebar collapses to icons on smaller screens / user toggle

    **Auth Flow:**

    5. Create login.tsx: Minimal, Vercel-style login page.
       - Centered card with "BOS3000 Admin" heading
       - Email + password form using shadcn Input + Button
       - On submit: call api.auth.Login({email, password}). On success, redirect to `/`. On error, show Chinese error message via error.ts mapping.
       - Auth guard: In layout.tsx, use a TanStack Query `useAuth()` hook that calls a `/auth/me` type endpoint. If 401, redirect to `/login`.

    **API Layer:**

    6. Create lib/api/client.ts: Placeholder for Encore-generated client. Export a class with the same interface structure so hooks compile. When Encore backend runs, replace with `encore gen client --lang=typescript --output=./src/lib/api/client.ts`.

    7. Create lib/api/hooks.ts: TanStack Query hook wrappers:
       - `useAuth()` -- query ['auth', 'me'], calls client auth check
       - `useLogin()` -- mutation for login
       - `useDashboardOverview()` -- query ['dashboard', 'overview'], refetchInterval: 30000
       - `useCustomers(params)` -- query ['customers', params]
       - `useTopUpCustomer()` -- mutation with invalidation of ['customers']
       - Additional hooks as stubs for: gateways, cdr, wastage, finance, did, compliance, ops, settings
       Each hook follows the pattern from RESEARCH.md.

    8. Create lib/api/error.ts: ERROR_MESSAGES record mapping English error codes to Chinese messages (INSUFFICIENT_BALANCE -> '...',  BLACKLISTED_NUMBER -> '...', etc.). Export getErrorMessage(code) and a toastError(err) helper using sonner.

    **Shared Components:**

    9. Create data-table.tsx: Generic DataTable<TData> component:
       - Props: columns (ColumnDef[]), data, totalCount, pagination (PaginationState), onPaginationChange, sorting (SortingState), onSortingChange, isLoading, toolbar (ReactNode for filters)
       - Uses useReactTable with getCoreRowModel, manualPagination, manualSorting
       - Renders shadcn Table with header row (sortable column headers), body rows, and pagination controls (page size selector, prev/next buttons, "X of Y" indicator)
       - When isLoading, show Skeleton rows instead of data
       - Chinese labels for pagination ("X 条/页", "共 X 条", "上一页", "下一页")

    10. Create stat-card.tsx: Reusable metric card component.
        - Props: title (string), value (string|number), description (string), icon (LucideIcon), trend (up/down/neutral + percentage), loading (boolean)
        - Renders shadcn Card with icon, title, large value, description, trend indicator
        - Loading state shows Skeleton

    11. Create confirm-dialog.tsx: Danger action confirmation dialog.
        - Props: title, description, confirmText, onConfirm, variant (danger/warning)
        - Uses shadcn AlertDialog with red styling for danger variant
        - Chinese button labels ("确认", "取消")

    12. Create csv-upload.tsx: CSV/Excel upload with preview.
        - Props: columns (expected column names), onImport (data[]), accept (".csv,.xlsx")
        - File input -> parse with Papa Parse (CSV) or SheetJS (xlsx) -> preview DataTable (first 10 rows) -> validate -> confirm button
        - Show validation errors inline (missing columns, invalid format)
        - Chinese labels throughout

    13. Create skeleton-page.tsx: Full-page loading skeleton.
        - Renders shadcn Skeleton blocks mimicking a typical page layout (header bar, stat cards row, table area)
