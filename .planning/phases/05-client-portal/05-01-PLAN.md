<task type="auto">
  <name>Task 1: Implement WebSocket Hub, handler, and message types in callback service</name>
  <files>
    callback/ws_hub.go, callback/ws_handler.go, callback/ws_message.go, callback/ws_hub_test.go
  </files>
  <action>
    1. Run `go get github.com/gorilla/websocket` in the project root.

    2. Create callback/ws_message.go:
       - Define `CallStatusEvent` struct with fields: CallID, UserID, Status, ALeg, BLeg, DurationSec (int), Timestamp (int64), all with json tags.
       - Status values: "initiating", "a_dialing", "a_connected", "b_dialing", "bridged", "finished", "failed".

    3. Create callback/ws_hub.go:
       - `Hub` struct: mu (sync.RWMutex), clients map[string]map[*Client]bool (user_id -> set), adminConns map[*Client]bool, broadcast chan *CallStatusEvent, register chan *Client, unregister chan *Client.
       - `Client` struct: hub *Hub, conn *websocket.Conn, send chan []byte, userID string, isAdmin bool.
       - `NewHub()` constructor initializing all channels and maps.
       - `Hub.Run()` goroutine: select on register (add to appropriate map), unregister (remove + close send), broadcast (json.Marshal event, send to all adminConns, send to clients[event.UserID]).
       - `Client.writePump()`: reads from send channel, writes to conn with WriteDeadline (10s), sends periodic pings (54s interval).
       - `Client.readPump()`: sets ReadDeadline (60s), handles pong to reset deadline, reads messages (discard -- server-to-client only), on error triggers unregister.
       - Export `Hub.Broadcast(event *CallStatusEvent)` method that sends to the broadcast channel -- this is what the state machine will call.

    4. Create callback/ws_handler.go:
       - Define Encore raw endpoint: `//encore:api public raw path=/ws/calls`
       - Handler signature: `func (s *Service) WebSocketHandler(w http.ResponseWriter, r *http.Request)`
       - Extract JWT from query param `?token=xxx` (browsers cannot set headers on WS upgrade).
       - Validate token using the same JWT validation logic from auth service. For now, create a `validateWSToken(token string) (userID string, isAdmin bool, err error)` method on Service that decodes the JWT and checks claims. If the token has an "admin" role claim, isAdmin=true.
       - Upgrade connection using gorilla/websocket Upgrader with CheckOrigin returning true (configure per env later).
       - Create Client, register with Hub, spawn writePump and readPump goroutines.
       - On validation failure, return http.StatusUnauthorized.

    5. Create callback/ws_hub_test.go:
       - Test NewHub creates properly initialized hub.
       - Test register: Create a mock client, send to register channel, verify it appears in the correct map (adminConns for admin, clients[userID] for regular).
       - Test broadcast routing: Register one admin and two regular clients (different user_ids), broadcast an event for user A, verify admin receives it, user A receives it, user B does NOT receive it.
       - Test unregister: Register then unregister a client, verify removed from maps.
       - Use goroutines and channels for synchronization in tests. Run Hub.Run() in a test goroutine.

    NOTE: The Hub must be initialized in the callback Service's initService() and Run() started as a goroutine. Add Hub field to the Service struct. The state machine (from Phase 2) will call hub.Broadcast() on state transitions -- that wiring happens naturally when the state machine code runs.
  </action>

  </verify>

    - Client readPump/writePump handle ping/pong lifecycle
