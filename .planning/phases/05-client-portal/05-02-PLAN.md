  <action>
    **App Shell:**

    1. Create main.tsx: Render root with Providers.

    2. Create providers.tsx:
       - QueryClientProvider with QueryClient (staleTime: 5min, retry: 1, refetchOnWindowFocus: false)
       - ThemeProvider: matchMedia('prefers-color-scheme: dark'), localStorage, `.dark` class on html. Export useTheme() hook.
       - RouterProvider with router

    3. Create nav-config.ts with sidebar navigation groups (all labels in Chinese):
       - 概览: 仪表盘 (LayoutDashboard, /)
       - 业务: 回拨操作 (PhoneCall, /callback), 话单查询 (FileText, /cdr), 损耗分析 (TrendingDown, /wastage)
       - 账务: 财务中心 (Wallet, /finance)
       - 开发: API 集成 (Code, /api-integration)
       - 设置: 账户设置 (Settings, /settings)

    4. Create router.tsx using React Router v7 library mode:
       - `createBrowserRouter` (import from `react-router`, NOT react-router-dom)
       - `/login` -> LoginPage
       - `/` -> Layout with lazy-loaded children (Suspense + SkeletonPage fallback):
         - index -> Dashboard, /callback -> Callback, /cdr -> CDR, /wastage -> Wastage
         - /finance -> Finance, /api-integration -> ApiIntegration, /settings -> Settings
       - Create placeholder components for each route (just `<div className="p-6">Feature Name</div>`)

    5. Create layout.tsx using shadcn Sidebar:
       - Left collapsible sidebar rendering nav-config groups
       - Top nav: Logo text "BOS3000 客户门户", spacer, theme toggle (Sun/Moon), user avatar dropdown (logout)
       - Below topnav: WSStatusBanner (shows when WebSocket disconnected)
       - Main content: `<Outlet />`
       - Sidebar collapses to icons on smaller screens / user toggle
       - Call useCallStatusWebSocket() at this level, pass isConnected to WSStatusBanner

    **Auth Flow:**

    6. Create login.tsx: Centered card with "BOS3000 客户门户" heading.
       - Email + password form using shadcn Input + Button + react-hook-form + Zod validation
       - On submit: call auth login API. On success, store JWT token in localStorage, redirect to `/`.
       - On error, show Chinese error message via error.ts.

    7. Create lib/api/client.ts:
       - Placeholder Encore client (same interface structure as Admin but for customer-facing APIs).
       - Wrap window.fetch globally to inject `Authorization: Bearer <token>` from localStorage before any API call.
       - Export client instance.

    8. Create lib/api/hooks.ts: TanStack Query hooks:
       - `useAuth()` -- query ['auth', 'me'], checks stored token validity
       - `useLogin()` -- mutation, stores token on success
       - `useLogout()` -- clears localStorage token, redirects to /login
       - `useDashboardOverview()` -- query ['dashboard', 'overview'], refetchInterval: 30000
       - `useActiveCalls()` -- query ['calls', 'active'] (WebSocket updates this cache)
       - Stubs for: useCDRList, useCDRDetail, useWastageOverview, useTransactions, useBalance, useApiKeys, useWebhookConfig, useAccountSettings

    9. Create lib/api/error.ts: ERROR_MESSAGES mapping English codes to Chinese messages. Export getErrorMessage(code) and toastError(err) using sonner.

    **WebSocket Hook:**

    10. Create hooks/use-call-ws.ts following the exact pattern from RESEARCH.md:
        - Connect to `${protocol}//${host}/ws/calls?token=${token}` (reads token from localStorage)
        - onopen: setIsConnected(true), reset reconnect counter
        - onmessage: Parse CallStatusEvent, update ['calls', 'active'] query cache via setQueryData (add new, update existing, remove finished/failed). Invalidate ['dashboard'] on call completion.
        - onclose: setIsConnected(false), exponential backoff reconnect (1s, 2s, 4s, 8s, 16s, max 30s)
        - Return { isConnected }
        - Cleanup: close WS and clear timeout on unmount
        - Do NOT connect if no token in localStorage

    11. Create components/shared/ws-status-banner.tsx:
        - Shows amber/destructive banner "实时连接已断开，正在重连..." when isConnected=false
        - Returns null when connected
        - Uses AlertTriangle icon
