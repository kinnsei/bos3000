    - Two concurrent PreDeduct calls for same user: one succeeds, one may fail if balance is insufficient after the first (serialized by row lock)

    - AcquireSlot when at max_concurrent: returns error with code ResourceExhausted, biz_code CONCURRENCY_LIMIT_EXCEEDED
    - ReleaseSlot(userID): decrements Redis counter
    - Finalize(callID, actualDurationSec): calculates A-leg cost (6s blocks) + B-leg cost (60s blocks), compares with pre-deduction, refunds excess or charges deficit, writes transaction record
    - Finalize with 0 duration (failed call): refunds full pre-deduction amount
  </behavior>

Create `billing/balance.go`:
- `//encore:api private method=POST path=/billing/pre-deduct` -- PreDeduct:
  - Accepts {UserID, CallID, ALegRate, BLegRate Money}
  - BEGIN transaction, SELECT balance FROM billing_accounts WHERE user_id=$1 FOR UPDATE
  - Calculate pre-deduction: (ALegRate + BLegRate) * 30 * 60 / 60 (30 min at combined rate)
  - Check balance + credit_limit >= preDeductAmount, else return INSUFFICIENT_BALANCE
  - UPDATE balance, INSERT into transactions (type=pre_deduct)
  - Return {Amount Money, TxID int64}

- `//encore:api private method=POST path=/billing/acquire-slot` -- AcquireSlot:
  - Accepts {UserID, MaxConcurrent int}
  - Redis INCR on concurrent slot key
  - If count > MaxConcurrent: decrement back, return CONCURRENCY_LIMIT_EXCEEDED (429)
  - Return {CurrentSlots int}

- `//encore:api private method=POST path=/billing/release-slot` -- ReleaseSlot:
  - Accepts {UserID}
  - Redis DECR, floor at 0
  - Return success

- `//encore:api private method=POST path=/billing/finalize` -- Finalize:
  - Accepts {UserID, CallID, ALegDurationSec, BLegDurationSec int64, ALegRate, BLegRate Money, PreDeductAmount Money}
  - Calculate A-leg cost: CeilDiv(ALegDurationSec, 6) * 6 * ALegRate / 60
  - Calculate B-leg cost: CeilDiv(BLegDurationSec, 60) * 60 * BLegRate / 60
  - actualCost = aLegCost + bLegCost
  - diff = PreDeductAmount - actualCost
  - If diff > 0: refund diff to balance, write transaction (type=refund)
  - If diff < 0: charge additional, write transaction (type=finalize)
  - If diff == 0: write transaction (type=finalize) with amount=0
  - Return {ALegCost, BLegCost, TotalCost, Refund Money}

Create `billing/migrations/2_create_rate_plans.up.sql`:
```sql
CREATE TABLE rate_plans (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    mode VARCHAR(20) NOT NULL DEFAULT 'uniform' CHECK (mode IN ('uniform', 'prefix')),
    uniform_a_rate BIGINT,
    uniform_b_rate BIGINT,
    description TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE rate_plan_prefixes (
    id BIGSERIAL PRIMARY KEY,
    rate_plan_id BIGINT NOT NULL REFERENCES rate_plans(id) ON DELETE CASCADE,
    prefix VARCHAR(10) NOT NULL,
    a_rate BIGINT NOT NULL,
    b_rate BIGINT NOT NULL,
    UNIQUE(rate_plan_id, prefix)
);
CREATE INDEX idx_rate_plan_prefixes_plan ON rate_plan_prefixes(rate_plan_id);
```

  - Accepts {Name, Mode, UniformARate, UniformBRate *Money, Description}

  - Check auth.Data().Role == "admin", else return PermissionDenied

- `//encore:api auth method=POST path=/billing/rate-plans/:id/prefixes` -- AddPrefixRate (admin only):
  - Accepts {Prefix, ARate, BRate Money}
  - INSERT into rate_plan_prefixes

- `//encore:api auth method=DELETE path=/billing/rate-plans/:id/prefixes/:prefix` -- RemovePrefixRate (admin only)

- `//encore:api private method=POST path=/billing/resolve-rate` -- ResolveRate:
  - Accepts {UserID int64, CalledPrefix string}
  - Step 1: SELECT a_leg_rate, b_leg_rate FROM billing_accounts WHERE user_id=$1
  - If both non-null: return user-level rates (highest priority per BILL-04)
  - Step 2: SELECT rate_plan_id FROM billing_accounts WHERE user_id=$1
  - If plan exists and mode=uniform: return uniform rates
  - If plan exists and mode=prefix: SELECT a_rate, b_rate FROM rate_plan_prefixes WHERE rate_plan_id=$1 AND prefix=$2
  - Step 3: If no rate found, return PREFIX_NOT_FOUND error
  - Return {ALegRate, BLegRate Money, Source string} (source = "user" | "plan_uniform" | "plan_prefix")

- `//encore:api auth method=PUT path=/billing/accounts/:userId/rate-config` -- SetUserRateConfig (admin only):
  - Accepts {RatePlanID *int64, ALegRate, BLegRate *Money}
  - UPDATE billing_accounts
