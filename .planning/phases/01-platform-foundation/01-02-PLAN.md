- Package `auth`, import sqldb, jwt/v5, bcrypt

- `//encore:service` annotation

Create `auth/handler.go`:
- `//encore:authhandler` on Service method
- Dispatch order: session cookie -> Bearer token -> API key (query)
- validateSessionCookie: parse JWT from cookie value, extract claims, verify role=admin, return AuthData
- validateBearerToken: parse JWT from "Bearer xxx", extract claims, return AuthData
- validateAPIKey: hash with SHA-256, lookup in api_keys table, check status=active, check IP whitelist (if configured), return AuthData with user's role
- IP whitelist check: parse request IP from encore.CurrentRequest(), compare against stored CIDR/exact IPs. If whitelist is empty, allow all. Use `errcode.IPNotWhitelisted` on failure.

Create `auth/login.go`:
- `//encore:api public method=POST path=/auth/admin/login` -- AdminLogin: accepts {email, password}, validates with bcrypt, creates JWT with role=admin claim, returns raw endpoint to set HttpOnly cookie
- Actually, Encore structured endpoints can't set cookies directly. Use `//encore:api public raw method=POST path=/auth/admin/login` to set Set-Cookie header.
- `//encore:api public method=POST path=/auth/client/login` -- ClientLogin: accepts {email, password}, validates with bcrypt, creates JWT with role=client claim, returns {token: "xxx", expires_at: "xxx"} in response body
- JWT claims: sub=user_id, role=admin|client, username, exp=24h, iat=now
- Use golang-jwt/jwt/v5 with HS256 signing

Create `auth/migrations/2_create_api_keys.up.sql`:
```sql
CREATE TABLE api_keys (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES users(id),
    prefix VARCHAR(20) NOT NULL,
    key_hash VARCHAR(64) NOT NULL UNIQUE,
    status VARCHAR(20) NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'revoked')),
    ip_whitelist TEXT[] DEFAULT '{}',
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_used_at TIMESTAMPTZ
);
CREATE INDEX idx_api_keys_hash ON api_keys(key_hash);
CREATE INDEX idx_api_keys_user ON api_keys(user_id);
```

Create `auth/apikey.go`:
- `//encore:api auth method=POST path=/auth/api-keys` -- CreateAPIKey: generate key with crypto/rand (32 bytes), prefix "bos_" + base64url, store SHA-256 hash + prefix. Only the owning user (auth.UserID match) can create. Return {key: "bos_xxx", prefix: "bos_xxx...", id: 123}.
- `//encore:api auth method=GET path=/auth/api-keys` -- ListAPIKeys: return [{id, prefix, status, created_at, last_used_at}] for current user. Never return full key or hash.
- `//encore:api auth method=POST path=/auth/api-keys/:id/reset` -- ResetAPIKey: generate new key, update hash+prefix, return new full key once.
- `//encore:api auth method=DELETE path=/auth/api-keys/:id` -- RevokeAPIKey: set status=revoked. Verify ownership (user_id = auth.UserID).
- `//encore:api auth method=POST path=/auth/api-keys/:id/ips` -- AddIPWhitelist: add IP/CIDR to ip_whitelist array.
- `//encore:api auth method=DELETE path=/auth/api-keys/:id/ips` -- RemoveIPWhitelist: remove IP from array.
- `//encore:api auth method=GET path=/auth/api-keys/:id/ips` -- ListIPWhitelist: return ip_whitelist array.
- All endpoints enforce ownership: WHERE user_id = currentUserID (from auth.Data()).
