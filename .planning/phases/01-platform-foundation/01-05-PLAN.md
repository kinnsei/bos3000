Create `compliance/migrations/1_create_blacklist.up.sql`:
```sql
CREATE TABLE blacklisted_numbers (
    id BIGSERIAL PRIMARY KEY,
    number VARCHAR(20) NOT NULL,
    user_id BIGINT,
    reason TEXT,
    created_by BIGINT NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE(number, user_id)
);
CREATE INDEX idx_blacklist_number ON blacklisted_numbers(number);
CREATE INDEX idx_blacklist_user ON blacklisted_numbers(user_id);
```
Note: user_id NULL = global blacklist. UNIQUE(number, user_id) uses NULLS NOT DISTINCT (PG 15+) or handle in application logic.

Create `compliance/blacklist.go`:
- `//encore:api private method=POST path=/compliance/check-blacklist` -- CheckBlacklist:
  - Accepts {CalledNumber string, UserID int64}
  - Query 1: SELECT EXISTS(SELECT 1 FROM blacklisted_numbers WHERE number = $1 AND user_id IS NULL)
  - If global hit: return BLACKLISTED_NUMBER error with message "number is globally blacklisted"
  - Query 2: SELECT EXISTS(SELECT 1 FROM blacklisted_numbers WHERE number = $1 AND user_id = $2)
  - If client hit: return BLACKLISTED_NUMBER error with message "number is blacklisted for this account"
  - Otherwise return nil

- `//encore:api auth method=POST path=/compliance/blacklist` -- AddBlacklist (admin or client for own):
  - Accepts {Number string, UserID *int64, Reason string}
  - Admin can add global (UserID=nil) or client-level
  - Client can only add to own blacklist (UserID must match auth.UserID)
  - INSERT INTO blacklisted_numbers

- `//encore:api auth method=DELETE path=/compliance/blacklist/:id` -- RemoveBlacklist:
  - Admin: any entry. Client: own entries only.

- `//encore:api auth method=GET path=/compliance/blacklist` -- ListBlacklist:
  - Query params: user_id filter (admin), global_only bool
  - Admin sees all; client sees own + global
