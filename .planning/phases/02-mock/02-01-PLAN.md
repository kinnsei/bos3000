**callback/fsclient/mock.go:**
- `MockConfig` struct with fields: ALegResult (string: "answer"/"no_answer"/"reject"/"error"), BLegResult (string: "answer"/"reject"/"error"/"busy"), BridgeResult (string: "stable"/"early_hangup"), BridgeDuration (time.Duration, simulated bridge time for billing calc), ALegHangupCause (string), BLegHangupCause (string)
- `MockFSClient` struct with: config MockConfig, handlers map[string][]func(CallEvent) with sync.RWMutex, calls map[string]*mockCallState with sync.Mutex
- `mockCallState` tracks: uuid, callID, leg, active bool
- `NewMockFSClient(config MockConfig) *MockFSClient` constructor
- `OriginateALeg`: if ALegResult=="error" return error. Otherwise generate uuid "mock-a-{callID}", store in calls map, fire events in a goroutine based on ALegResult:
  - "answer": fire CHANNEL_ANSWER
  - "no_answer"/"reject": fire CHANNEL_HANGUP with mapped cause
- `OriginateBLegAndBridge`: if BLegResult is "error"/"reject"/"busy" return error with descriptive message. Otherwise generate uuid "mock-b-{callID}", fire events in goroutine:
  - "answer": fire B CHANNEL_ANSWER, then CHANNEL_BRIDGE
  - If BridgeResult=="early_hangup": after CHANNEL_BRIDGE, fire CHANNEL_HANGUP quickly
  - If BridgeResult=="stable": after CHANNEL_BRIDGE, wait BridgeDuration (but since instant mock, just record the duration), fire CHANNEL_HANGUP
- `HangupCall`: find call in calls map, mark inactive, fire CHANNEL_HANGUP
- `StartRecording`, `StopRecording`: return nil (no-op for Phase 2)
- `RegisterEventHandler`: store handler in handlers map (thread-safe with RWMutex)
- `fireEvent(event CallEvent)`: iterate registered handlers for event.EventName, call each

**IMPORTANT:** Events must fire in a separate goroutine with a tiny yield (`runtime.Gosched()` or buffered channel) to avoid race condition where the state machine hasn't registered its event listener yet. Use `go func() { ... }()` pattern from RESEARCH.md.
