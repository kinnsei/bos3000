**callback/statemachine.go -- `func (s *Service) runCall(callCtx context.Context, call *CallbackCall)`:**
The state machine runs as a goroutine per call. Uses a context derived from callCtx (which is cancellable for force hangup). Steps:

1. **A-leg originate**: Call `s.fsClient.OriginateALeg(ctx, params)`. If error, finalizeCall with reason "a_originate_failed", return. Set call.AfsUUID, update status "a_dialing", set call.ADialAt=now.

2. **Wait for A-leg event**: Read from activeCall.eventCh with select:
   - `case event := <-ac.eventCh`: if CHANNEL_ANSWER on leg A, proceed. If CHANNEL_HANGUP on leg A, finalizeCall with A hangup cause (no wastage -- A never connected if cause is before answer, or a_failed).
   - `case <-parkTimer.C`: Park timeout (read park_timeout_sec from system_configs, default 60s). Call fsClient.HangupCall(aUUID, "ALLOTTED_TIMEOUT"). finalizeCall with reason "a_park_timeout". (CALL-02)
   - `case <-ctx.Done()`: Force hangup or shutdown. finalizeCall with reason "force_hangup".

3. **A-leg answered**: Update status "a_connected", set call.AAnswerAt=event.Timestamp.

4. **B-leg originate**: Call `s.fsClient.OriginateBLegAndBridge(ctx, aUUID, bParams)`. If error (command-level failure per CALL-03):
   - Call fsClient.HangupCall(aUUID, "NORMAL_CLEARING") to hang up A
   - Mark wastage: call was a_connected but B never answered
   - finalizeCall with reason "b_originate_failed"
   - return

5. Set call.BfsUUID, update status "b_dialing", set call.BDialAt=now.

6. **Wait for B-leg events**: Loop reading from eventCh:
   - B CHANNEL_ANSWER: update status "b_connected", set call.BAnswerAt
   - CHANNEL_BRIDGE: update status "bridged", set call.BridgeAt
   - CHANNEL_HANGUP (any leg): break out of loop

7. **Wait for call end (bridged state)**: If bridged, wait for CHANNEL_HANGUP or max_duration timeout:
   - `case event := <-ac.eventCh`: CHANNEL_HANGUP received. Set hangup info.
   - `case <-maxDurationTimer.C`: Call fsClient.HangupCall for both legs. Set hangup_by="system".
   - `case <-ctx.Done()`: Force hangup.

8. Set bridge_end_at, calculate bridge_duration_ms. finalizeCall.

**CRITICAL per CONTEXT.md:** Do NOT use defer for concurrent slot release. finalizeCall is the single release point. Use sync.Once on activeCall.finalized to prevent double finalization (Pitfall 3 from RESEARCH.md).

**CRITICAL per CONTEXT.md:** B-leg originate failure does NOT retry with backup gateway. Direct failure.

**callback/finalize.go -- `func (s *Service) finalizeCall(call *CallbackCall, reason string, originErr error)`:**
Uses activeCall.finalized sync.Once to ensure single execution.

1. Calculate A-leg duration: if AAnswerAt set, ADurationMs = AHangupAt - AAnswerAt (or now if not yet set)
2. Calculate B-leg duration: if BAnswerAt set, BDurationMs = BHangupAt - BAnswerAt
3. Calculate bridge duration: if BridgeAt set, BridgeDurationMs = BridgeEndAt - BridgeAt
4. Read bridge_broken_early_threshold_sec from system_configs (default 10)
5. Call `classifyWastage(call, threshold)` to get wastageType, wastageCost
6. Set call fields: WastageType, WastageCost, WastageDurationMs
7. Determine final status: if reason indicates success (normal hangup after bridge with sufficient duration) -> "finished", else -> "failed"
8. Call Phase 1 billing.Finalize to settle actual costs and refund pre-deduction excess
9. Call Phase 1 billing.ReleaseSlot to free concurrent slot (NOT using defer per PROJECT.md decision)
10. Call updateCallFinal to persist all final state to DB
11. Remove from s.activeCalls map

**NOTE on Phase 1 dependency:** Since Phase 1 is not yet executed, the imports for billing/routing/compliance services won't exist yet. Use conditional compilation or create placeholder import stubs. The recommended approach: create the function calls as they should be, and the code will compile once Phase 1 is implemented. If needed for isolated testing, create a thin mock layer. However, since `encore test` requires all services to exist, the executor should verify if bare stubs are needed and create minimal ones if compilation fails.

**callback/wastage.go -- `func classifyWastage(call *CallbackCall, bridgeBrokenThresholdSec int64) (string, int64)`:**
Follow the exact logic from RESEARCH.md:
- Case 1 (a_connected_b_failed): AAnswerAt != nil AND BAnswerAt == nil. WastageCost = A-leg cost during wait (ceil to seconds, apply 6s billing blocks with ALegRate).
- Case 2 (bridge_broken_early): BridgeAt != nil AND BridgeEndAt != nil AND bridgeDurationSec < threshold. WastageCost = ALegCost + BLegCost for the short bridge period.
- Default: no wastage, return "", 0.

Cost calculation helper: `calculateCost(durationSec, ratePerMin int64, blockSizeSec int64) int64` -- rounds up to block size, calculates cost in fen. Formula: blocks = ceil(durationSec / blockSizeSec), cost = blocks * blockSizeSec * ratePerMin / 60.
