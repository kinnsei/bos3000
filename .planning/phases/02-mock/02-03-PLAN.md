**callback/initiate.go -- POST /callbacks:**
```
//encore:api auth method=POST path=/callbacks
func (s *Service) InitiateCallback(ctx context.Context, p *InitiateCallbackParams) (*InitiateCallbackResponse, error)
```

`InitiateCallbackParams` struct:
- ANumber string `json:"a_number"` -- required
- BNumber string `json:"b_number"` -- required
- CallerID *string `json:"caller_id,omitempty"` -- optional: specify outgoing caller ID
- MaxDuration *int `json:"max_duration,omitempty"` -- optional: max call seconds, default from system_configs
- CustomData json.RawMessage `json:"custom_data,omitempty"` -- optional: transparent JSON, max 1KB
- PreferredGateway *string `json:"preferred_gateway,omitempty"` -- optional: preferred gateway name
- CallbackURL *string `json:"callback_url,omitempty"` -- optional: webhook URL for this call

`Validate() error` method:
- a_number and b_number must be non-empty
- custom_data if present must be valid JSON and <= 1024 bytes
- max_duration if present must be > 0

`InitiateCallbackResponse` struct: CallID string, Status string

Flow (per RESEARCH.md Initiate Callback API example):
1. Get auth data from context (auth.Data) to get UserID, Role, MaxConcurrent, DailyLimit
2. Call compliance.CheckBlacklist for b_number -- if blocked, return PermissionDenied
3. Call compliance.CheckDailyLimit -- if exceeded, return ResourceExhausted (429)
4. Call billing.ResolveRate with b_number prefix (first 3 chars) -- get ALegRate, BLegRate
5. Generate callID with uuid.New().String()
6. Call billing.PreDeduct -- if insufficient balance, return error (402 from billing)
7. Call billing.AcquireSlot -- if exceeded, refund pre-deduction via billing.Finalize (zero duration), return ResourceExhausted (429)
8. Call routing.PickALeg (pass preferred_gateway if specified)
9. Call routing.PickBLeg with b_number
10. Call routing.SelectDID for caller ID (use explicit caller_id param if provided, else DID from routing)
11. Create CallbackCall struct with all fields, status="initiating"
12. INSERT into DB via insertCall
13. Create activeCall with context, event channel (buffered 10), register in activeCalls map
14. Launch `go s.runCall(callCtx, call)` -- state machine goroutine
15. Return {CallID, Status: "initiating"}

**callback/hangup.go -- POST /callbacks/:id/hangup:**
```
//encore:api auth method=POST path=/callbacks/:id/hangup
func (s *Service) ForceHangup(ctx context.Context, id string) (*ForceHangupResponse, error)
```

`ForceHangupResponse` struct: CallID string, Status string, Message string

Flow:
1. Get auth data (UserID, Role)
2. Lookup call in DB to verify existence and ownership
3. If role=client AND call.UserID != auth.UserID, return PermissionDenied
4. If call.Status in ("finished", "failed"), return error "call already terminated"
5. Lookup in activeCalls map. If found, call ac.cancel() to cancel state machine context
6. Also call fsClient.HangupCall for A-leg UUID (and B-leg if exists)
7. Set hangup_by based on role: "admin" or "client"
8. Return {CallID, Status: "finished", Message: "Call terminated"}

Per CONTEXT.md: force hangup is treated as normal end, billed by actual duration.
