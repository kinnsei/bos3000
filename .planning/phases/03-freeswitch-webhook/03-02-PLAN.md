  4. Capture stderr for error reporting
  5. Return nil on success, wrapped error on failure

**recording/worker.go:**
- Pub/Sub subscription:
  ```go
  var _ = pubsub.NewSubscription(
      RecordingMergeTopic, "recording-merge-worker",
      pubsub.SubscriptionConfig[*RecordingMergeEvent]{
          Handler:   pubsub.MethodHandler((*Service).HandleMerge),
          RetryPolicy: &pubsub.RetryPolicy{
              MinBackoff: 30 * time.Second,
              MaxBackoff: 5 * time.Minute,
              MaxRetries: 3,
          },
      },
  )
  ```
- `HandleMerge(ctx context.Context, event *RecordingMergeEvent) error`:
  1. Build output path: `/tmp/{callID}_merged.mp3`
  2. Call MergeRecordings(ctx, event.AFilePath, event.BFilePath, outputPath)
  3. If merge fails, update callback_calls recording_status='failed', return error (triggers Pub/Sub retry)
  4. Upload merged file to S3: key = `recordings/{customerID}/{date}/{callID}_merged.mp3`
  5. Upload A-leg WAV: key = `recordings/{customerID}/{date}/{callID}_a.wav`
  6. Upload B-leg WAV: key = `recordings/{customerID}/{date}/{callID}_b.wav`
  7. Update callback_calls: SET recording_key, recording_a_key, recording_b_key, recording_status='ready'
  8. Clean up local merged file (os.Remove). Original WAVs cleaned by 24h cron.
  9. Return nil

- Use `sqldb.Named("callback")` to access callback service's database for updating recording columns (shared DB access per Encore pattern)

**recording/api.go:**

Create two API endpoints:

1. **GET /recordings/url** (auth required):
```go
type GetRecordingURLParams struct {
    CallID string `query:"call_id"`
    Type   string `query:"type"` // "merged", "a", "b"
}

type GetRecordingURLResponse struct {
    URL       string    `json:"url"`
    ExpiresAt time.Time `json:"expires_at"`
}

//encore:api auth method=GET path=/recordings/url
func (s *Service) GetRecordingURL(ctx context.Context, p *GetRecordingURLParams) (*GetRecordingURLResponse, error)
```
- Validate Type is one of "merged", "a", "b" (return InvalidArgument otherwise)
- Query callback_calls for the call by call_id using sqldb.Named("callback")
- Check auth: use `auth.UserID()` and `auth.Data()` to verify:
  - Admin role: can access any recording
  - Client role: can only access own recordings (WHERE user_id = current_user_id)
- Look up the appropriate recording key based on Type:
  - "merged" -> recording_key
  - "a" -> recording_a_key
  - "b" -> recording_b_key
- If recording_status != 'ready' or key is NULL, return NotFound "recording not available"
- Generate presigned URL: `RecordingsBucket.SignedDownloadURL(ctx, key, objects.WithTTL(15 * time.Minute))`
- Return URL + ExpiresAt (now + 15 min)
